# Security Policy

## Supported Versions

We only provide security updates for the current active development branch of ZapOS.

| Version | Supported          |
| ------- | ------------------ |
| Pre-Alpha | :white_check_mark: |
| < 0.1.0 | :x:                |

## Reporting a Vulnerability

**Do not report security vulnerabilities through public GitHub issues.** If you find a bug that could compromise user data, system integrity, or performance stability, please report it directly to the lead developer.

### How to report:
1. **Discord:** DM `zapl_official` with the tag **[SECURITY REPORT]**.
2. **Email:** zapl.informatics@gmail.com

### Our Commitment:
* We will acknowledge your report within **48 hours**.
* We will provide a timeline for a fix.
* You will be credited in the **ZapOS Security Hall of Fame** (unless you wish to remain anonymous).

*Informatics > Luck. Let's build a secure core.*
